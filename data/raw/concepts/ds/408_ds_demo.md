# 数据结构基础概念（408 Demo）

## 1.1 线性表

线性表是一种具有相同数据类型的 n 个数据元素的有限序列，一般记为 (a1, a2, ..., an)。  
线性表的特点是：元素之间具有“一对一”的逻辑关系，除第一个元素外，每个元素都有且只有一个直接前驱；除最后一个元素外，每个元素都有且只有一个直接后继。

常见的线性表实现方式有：
- 顺序表：用一段连续的存储空间依次存放线性表中的元素。
- 链表：用一组任意的存储单元来存放线性表中的元素，通过指针字段表示元素之间的逻辑关系。

## 1.2 栈（Stack）

栈是一种受限的线性表，只允许在一端进行插入和删除操作。这一端被称为栈顶，另一端称为栈底。  
栈遵循“后进先出（LIFO）”的原则。

典型应用包括：
- 函数调用：通过调用栈保存返回地址和局部变量。
- 表达式求值与括号匹配。
- 深度优先搜索（DFS）中的结点保存。

## 1.3 队列（Queue）

队列也是一种受限的线性表，只允许在一端进行插入（入队），另一端进行删除（出队）。  
队列遵循“先进先出（FIFO）”的原则。

典型应用包括：
- 操作系统中的就绪队列、等待队列。
- 广度优先搜索（BFS）中的结点保存。
- 缓冲区管理（生产者-消费者模型）。

## 1.4 树与二叉树

树是一种非线性数据结构，用来表示“层次关系”。  
特点：
- 有一个特殊的结点称为根结点。
- 根结点之外的结点可分成若干棵互不相交的子树。
- 树中结点之间存在“多对一”的父子关系。

二叉树是每个结点最多只有两个孩子的树，通常称为左孩子和右孩子。

特殊的二叉树：
- 满二叉树：所有分支结点都有两个孩子，并且所有叶子结点在同一层。
- 完全二叉树：除了最后一层外，其余各层结点数都达到最大，最后一层的结点都集中在最左侧。

## 1.5 图（Graph）

图是由顶点集合和边集合组成的数据结构，常用于表示“多对多”的关系。  
根据边是否有方向，可以分为：
- 无向图
- 有向图

典型应用包括：
- 计算机网络中的路由
- 社交网络中的关系分析
- 拓扑排序、最短路径、最小生成树等问题
